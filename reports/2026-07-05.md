### K-1. Ethan Mollick (2026-06-30) "The Twilight of the Chatbots"

Mollick 提出聊天界面正在消亡，AI 能力加速的同时政府管控正在限制普通人接触最前沿模型的渠道，企业级 Agent 将取代消费者对话式交互，形成"能力越强、触达越窄"的悖论格局。
📌 *给企业的启发：提前布局 Agent 工作流而非停留在 Chatbot POC，因为 Chatbot 形态正被跳过*

### K-2. Simon Willison (2026-07-02) "Coding Agent 认知债务"

引用 Geoffrey Litt 框架，指出编程 Agent 生成的代码虽能通过测试，但会累积"认知债务"——开发者对代码理解的空白，长期维护成本远超短期效率收益，呼吁建立 Agent 生成代码的人工审查节奏。
📌 *给企业的启发：在 Agent 编程策略中硬编码"每周认知审计"流程，避免技术债务滚雪球*

### K-3. Cory Doctorow (2026-06-18) "AI 数字主权风险不存在"

Doctorow 撰文反驳"AI 主权"叙事，认为所谓"主权 AI"本质上是大厂以地缘政治话术锁定政府采购，真正的自主能力应建立在开源模型与本土算力上，而非购买商业 API 的本地化部署版本。
📌 *给企业的启发：评估"主权 AI"方案时区分真开源自主与换壳商业锁定*

### K-4. Cory Doctorow (2026-06-06) "The Reverse Centaur's Guide" 新书预告

Doctorow 新书探讨"反向人马"——人类沦为 AI 执行器的场景，以 Amazon 仓储工人为原型分析自动化决策如何将人类降格为肌肉接口，呼吁在 AI 部署中保留人类判断权。
📌 *给企业的启发：AI 部署审查应包含"反向人马检查"——员工是否被降格为纯执行层*
